# CONTEXTO DEL PROYECTO: EduBeta

## Características del Proyecto
EduBeta es una plataforma de gestión académica integral diseñada para instituciones educativas. Utiliza un stack moderno: React, TypeScript, Tailwind CSS y Firebase (Firestore, Auth, Storage).

### REGLA DE ORO GLOBAL
EduBeta es utilizada en tiempo real desde el aula por docentes, padres y coordinadores predominantemente en pantallas táctiles (móviles y tablets). Todo nuevo diseño o vista **DEBE ser 100% funcional y fluido en móvil**.

## Estándares de Diseño (ClassComponent.md / skill.md)
- **Carga Global**: Todas las esperas de datos deben mostrar el componente `<LoadingSpinner />`.
- **Identidad Visual (Avatares)**: Prioridad Foto (`avatar_url`) > Iniciales. Fondo `#C6E7FC`, Texto `#0099FE`.
- **Botones**: Uso de `EduButton` para consistencia. Bordes `rounded-[5px]`, sombras `shadow-primary/20`.
- **Formularios**: Uso de `FormView` (sin transiciones artificiales) para navegación instantánea.
- **Componentes**: Uso obligatorio de `EduInput` y `EduSelect` para campos de entrada.

## Estructura de Navegación y Permisos
La aplicación se organiza en 5 categorías principales:

1. **EduBeta**: Inicio (Acceso: Todos).
2. **Gestión Académica**: Falta, Estudiantes, Docentes, Grado, Asignatura, Barrios, Asignación Académica, Historial (Acceso: Admin y Coordinadores).
3. **Herramientas**: Tareas, Asistencia, Observador, Horario, ISA (Acceso: Admin, Coordinadores y Docentes).
4. **Aplicación**: Configuración (Acceso: Todos).
5. **Cliente**: Caracterización (Acceso: Solo Admin).

## Hitos Recientes (Marzo 2026)
- **Estandarización UI**: Implementación global de `EduButton`, `EduInput` y `EduSelect`.
- **Eliminación de Transiciones**: Retirada de estados `exiting` y retardos `setTimeout` para una navegación instantánea.
- **Limpieza de Interfaz**: Eliminación de botones "Cancelar" en favor de navegación nativa/Breadcrumbs.
- **Acceso Docente**: Ampliación de permisos para que el rol `teacher` vea su carga académica.
